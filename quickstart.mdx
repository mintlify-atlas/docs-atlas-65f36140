title: "Quickstart"
description: "Build your first JReactive component in under five minutes"
---

This guide walks you through creating a Spring Boot application with a live-updating counter component. By the end, you will have a working JReactive app running locally.

<Note>
  JReactive is distributed through [JitPack](https://jitpack.io). You need a JitPack repository entry in your build file. All artifacts are under the `com.ciro` group ID at version `0.0.1-SNAPSHOT`.
</Note>

<Steps>

<Step title="Add the JitPack repository">

Add the JitPack repository to your Maven `pom.xml` so Maven can resolve JReactive artifacts:

```xml pom.xml
<repositories>
  <repository>
    <id>jitpack.io</id>
    <url>https://jitpack.io</url>
  </repository>
</repositories>
```

</Step>

<Step title="Add dependencies">

Add the following dependencies. `jreactive-starter-spring` pulls in Spring Boot web and WebSocket support. `jreactive-apt` runs at compile time only. `jreactive-runtime-js` provides the browser-side JavaScript runtime. `jreactive-ui` is optional but includes ready-made components.

<CodeGroup>

```xml Maven
<properties>
  <java.version>21</java.version>
  <spring-boot.version>3.2.5</spring-boot.version>
  <jreactive.version>0.0.1-SNAPSHOT</jreactive.version>
</properties>

<dependencies>
  <!-- Spring Boot auto-configuration + WebSocket -->
  <dependency>
    <groupId>com.ciro</groupId>
    <artifactId>jreactive-starter-spring</artifactId>
    <version>${jreactive.version}</version>
  </dependency>

  <!-- JVM runtime: protocol handler, HTTP API, page resolver -->
  <dependency>
    <groupId>com.ciro</groupId>
    <artifactId>jreactive-runtime-jvm</artifactId>
    <version>${jreactive.version}</version>
  </dependency>

  <!-- Browser-side JS runtime (SockJS, Idiomorph, jreactive-runtime.js) -->
  <dependency>
    <groupId>com.ciro</groupId>
    <artifactId>jreactive-runtime-js</artifactId>
    <version>${jreactive.version}</version>
  </dependency>

  <!-- Optional: built-in UI components (JButton, JInput, JTable, etc.) -->
  <dependency>
    <groupId>com.ciro</groupId>
    <artifactId>jreactive-ui</artifactId>
    <version>${jreactive.version}</version>
  </dependency>
</dependencies>
```

```groovy Gradle
repositories {
    maven { url 'https://jitpack.io' }
}

dependencies {
    implementation 'com.ciro:jreactive-starter-spring:0.0.1-SNAPSHOT'
    implementation 'com.ciro:jreactive-runtime-jvm:0.0.1-SNAPSHOT'
    implementation 'com.ciro:jreactive-runtime-js:0.0.1-SNAPSHOT'
    implementation 'com.ciro:jreactive-ui:0.0.1-SNAPSHOT'
}
```

</CodeGroup>

</Step>

<Step title="Configure the annotation processor">

`jreactive-apt` is the AOT annotation processor. It runs during `mvn compile` and generates zero-reflection accessor classes, pre-compiles client-side JS templates, and applies scoped CSS. Configure it in `maven-compiler-plugin`:

```xml pom.xml
<build>
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-compiler-plugin</artifactId>
      <configuration>
        <source>21</source>
        <target>21</target>
        <annotationProcessorPaths>
          <path>
            <groupId>com.ciro</groupId>
            <artifactId>jreactive-apt</artifactId>
            <version>${jreactive.version}</version>
          </path>
        </annotationProcessorPaths>
        <fork>true</fork>
        <compilerArgs>
          <arg>--add-exports</arg>
          <arg>jdk.compiler/com.sun.source.util=ALL-UNNAMED</arg>
          <arg>--add-exports</arg>
          <arg>jdk.compiler/com.sun.source.tree=ALL-UNNAMED</arg>
          <!-- Required for @Param injection by parameter name -->
          <arg>-parameters</arg>
        </compilerArgs>
      </configuration>
    </plugin>

    <plugin>
      <groupId>org.springframework.boot</groupId>
      <artifactId>spring-boot-maven-plugin</artifactId>
      <version>${spring-boot.version}</version>
      <executions>
        <execution>
          <goals><goal>repackage</goal></goals>
        </execution>
      </executions>
    </plugin>
  </plugins>
</build>
```

<Tip>
  Add `build-helper-maven-plugin` to include the generated sources directory so your IDE picks up accessor classes:

  ```xml
  <plugin>
    <groupId>org.codehaus.mojo</groupId>
    <artifactId>build-helper-maven-plugin</artifactId>
    <version>3.4.0</version>
    <executions>
      <execution>
        <id>add-source</id>
        <phase>generate-sources</phase>
        <goals><goal>add-source</goal></goals>
        <configuration>
          <sources>
            <source>${project.build.directory}/generated-sources/annotations</source>
          </sources>
        </configuration>
      </execution>
    </executions>
  </plugin>
  ```
</Tip>

</Step>

<Step title="Create the Spring Boot main class">

A standard `@SpringBootApplication` entry point. JReactive's auto-configuration (`JReactiveAutoConfiguration`) discovers and wires everything automatically.

```java src/main/java/com/example/myapp/MyApplication.java
package com.example.myapp;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class MyApplication {

    public static void main(String[] args) {
        SpringApplication.run(MyApplication.class, args);
    }
}
```

</Step>

<Step title="Create a layout">

A layout wraps every page. It provides the full HTML shell, navigation, and the three required script tags. Place a `<slot />` element where the page content should appear.

```java src/main/java/com/example/myapp/MainLayout.java
package com.example.myapp;

import com.ciro.jreactive.HtmlComponent;

public class MainLayout extends HtmlComponent {

    @Override
    protected String template() {
        return """
            <!DOCTYPE html>
            <html lang="en">
            <head>
                <meta charset="UTF-8">
                <meta name="viewport" content="width=device-width, initial-scale=1.0">
                <title>My JReactive App</title>
            </head>
            <body>
                <nav>
                    <strong>My App</strong>
                    <a href="/" data-router>Home</a>
                </nav>

                <main>
                    <slot />
                </main>

                <!-- Required: Idiomorph DOM morphing, SockJS transport, JReactive runtime -->
                <script src="/js/idiomorph.min.js"></script>
                <script src="/js/sockjs.min.js"></script>
                <script src="/js/jreactive-runtime.js"></script>
            </body>
            </html>
        """;
    }
}
```

<Note>
  The three `<script>` tags at the bottom of `<body>` are required. `idiomorph.min.js` performs DOM morphing, `sockjs.min.js` provides the WebSocket transport, and `jreactive-runtime.js` is the JReactive client runtime. All three are served automatically from `jreactive-runtime-js`.
</Note>

</Step>

<Step title="Create a base page class">

Create an abstract base that applies the layout to all pages using `@Layout`. Pages that extend this class automatically render inside `MainLayout`.

```java src/main/java/com/example/myapp/AppPage.java
package com.example.myapp;

import com.ciro.jreactive.HtmlComponent;
import com.ciro.jreactive.router.Layout;

@Layout(MainLayout.class)
public abstract class AppPage extends HtmlComponent {
    // Shared state or utility methods for all pages go here
}
```

</Step>

<Step title="Create a counter component">

This is the core JReactive pattern. Extend `HtmlComponent`, declare reactive fields with `@State`, and mark event handler methods with `@Call`. The `template()` method returns HTML with `{{variable}}` mustache bindings and `@click` event directives.

```java src/main/java/com/example/myapp/CounterPage.java
package com.example.myapp;

import com.ciro.jreactive.State;
import com.ciro.jreactive.annotations.Call;
import com.ciro.jreactive.annotations.StatefulRam;
import com.ciro.jreactive.router.Route;

@Route(path = "/")
@StatefulRam
public class CounterPage extends AppPage {

    @State
    public int count = 0;

    @Call
    public void increment() {
        count++;
    }

    @Call
    public void reset() {
        count = 0;
    }

    @Override
    protected String template() {
        return """
            <div>
                <h1>Counter</h1>
                <p>Current value: <strong>{{count}}</strong></p>
                <button @click="increment()">Increment</button>
                <button @click="reset()">Reset</button>
            </div>
        """;
    }
}
```

**What each annotation does:**

| Annotation | Location | Purpose |
|---|---|---|
| `@Route(path = "/")` | Class | Registers this component as the handler for the `/` URL |
| `@StatefulRam` | Class | Keeps component state in JVM memory (Caffeine cache); never serialized to Redis |
| `@State` | Field | Marks `count` as reactive; changes are automatically detected and pushed to the browser |
| `@Call` | Method | Exposes `increment()` and `reset()` as callable endpoints; invoked via `POST /call/{componentId.methodName}` |

<Warning>
  Every JReactive application must have exactly one `@Route(path = "/")` component. The `RouteRegistry` will throw an `IllegalStateException` at startup if no root route is registered.
</Warning>

</Step>

<Step title="Run the application">

```bash
mvn spring-boot:run
```

Open [http://localhost:8080](http://localhost:8080) in your browser. You should see the counter page. Clicking **Increment** updates the count in real time over WebSocket — no page reload, no JavaScript written by you.

**What happens on first load:**

1. Spring Boot handles the HTTP request and renders `CounterPage` inside `MainLayout`.
2. The browser receives the full HTML snapshot including the current `count` value.
3. The JReactive JS runtime opens a WebSocket connection and subscribes to binding updates.
4. When you click **Increment**, the browser posts to `/call/CounterPage-1.increment` (or just `increment` for the root component).
5. `JrxHttpApi` invokes `increment()`, the `count` field changes, and the updated value is pushed back over WebSocket.
6. The runtime applies the delta using Idiomorph DOM morphing.

</Step>

</Steps>
