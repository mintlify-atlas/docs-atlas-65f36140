title: "ComponentAccessor"
description: "Generated zero-reflection accessor interface. The APT processor creates a ClassName__Accessor implementation at compile time, eliminating Field.get/set and Method.invoke overhead at runtime."
---

`ComponentAccessor<T>` is the reflection-free access layer for JReactive components. At compile time, the `jreactive-apt` annotation processor generates a concrete implementation named `ClassName__Accessor` for each component class. The runtime uses this accessor instead of `java.lang.reflect` for all field reads, field writes, and method invocations.

**Package:** `com.ciro.jreactive.spi`

## Interface definition

```java
public interface ComponentAccessor<T extends HtmlComponent> {

    // Replaces Field.set()
    void write(T component, String property, Object value);

    // Replaces Field.get()
    Object read(T component, String property);

    // Replaces Method.invoke()
    Object call(T component, String method, Object... args);

    default String renderStatic(T component) { return null; }

    default List<JrxNode> getAst() { return null; }
    default String getScopedCss() { return null; }
}
```

## Methods

<ResponseField name="write" type="void">
  Sets the field named `property` on `component` to `value`. The generated implementation uses a direct field assignment (`component.myField = value`) with no reflection overhead.

  | Parameter | Type | Description |
  |---|---|---|
  | `component` | `T` | The component instance to modify |
  | `property` | `String` | The Java field name |
  | `value` | `Object` | The new value |
</ResponseField>

<ResponseField name="read" type="Object">
  Returns the current value of the field named `property` from `component`. The generated implementation is a direct field read.

  | Parameter | Type | Description |
  |---|---|---|
  | `component` | `T` | The component instance to read from |
  | `property` | `String` | The Java field name |
</ResponseField>

<ResponseField name="call" type="Object">
  Invokes the method named `method` on `component` with the provided `args`. The generated implementation dispatches directly to the concrete method by name, with a `switch` or `if-else` chain.

  | Parameter | Type | Description |
  |---|---|---|
  | `component` | `T` | The component instance |
  | `method` | `String` | The Java method name |
  | `args` | `Object...` | Arguments in declaration order |
</ResponseField>

<ResponseField name="renderStatic" type="String">
  Returns a pre-rendered static HTML string for the component, if the APT processor was able to generate one. Returns `null` when dynamic rendering is required. Default implementation returns `null`.
</ResponseField>

<ResponseField name="getAst" type="List&lt;JrxNode&gt;">
  Returns the pre-compiled abstract syntax tree (AST) of the component's template. Used by the client-side rendering pipeline to produce JS templates. Default implementation returns `null`.
</ResponseField>

<ResponseField name="getScopedCss" type="String">
  Returns scoped CSS extracted from the component template at compile time. Default implementation returns `null`.
</ResponseField>

## Generated class naming

For a component class `com.example.UserPage`, the APT processor generates:

```
com.example.UserPage__Accessor
```

The generated class registers itself in `AccessorRegistry` via a static initializer:

```java
// Generated by jreactive-apt
public final class UserPage__Accessor implements ComponentAccessor<UserPage> {

    static {
        AccessorRegistry.register(UserPage.class, new UserPage__Accessor());
    }

    @Override
    public void write(UserPage c, String property, Object value) {
        switch (property) {
            case "count" -> c.count = (Integer) value;
            case "label" -> c.label = (String) value;
            default -> throw new IllegalArgumentException("Unknown property: " + property);
        }
    }

    @Override
    public Object read(UserPage c, String property) {
        return switch (property) {
            case "count" -> c.count;
            case "label" -> c.label;
            default -> throw new IllegalArgumentException("Unknown property: " + property);
        };
    }

    @Override
    public Object call(UserPage c, String method, Object... args) {
        return switch (method) {
            case "increment" -> { c.increment(); yield null; }
            case "reset"     -> { c.reset();     yield null; }
            default -> throw new IllegalArgumentException("Unknown method: " + method);
        };
    }
}
```

## `AccessorRegistry`

`AccessorRegistry` is the runtime lookup table for accessor instances.

```java
public class AccessorRegistry {

    // Register an accessor (called from the generated static block)
    public static void register(Class<?> clazz, ComponentAccessor<?> accessor);

    // Look up an accessor; returns null if not found (reflection fallback)
    public static <T extends HtmlComponent> ComponentAccessor<T> get(Class<T> clazz);

    // Returns true if an accessor exists for the given class
    public static boolean has(Class<?> clazz);
}
```

### Lazy loading

`AccessorRegistry.get()` uses lazy class loading. If a class has not been loaded yet, `get()` attempts to load `ClassName__Accessor` via `Class.forName()`. Loading the accessor class triggers its `static {}` block, which calls `register()`. Subsequent lookups hit the cache directly.

If no accessor class is found (e.g., the class was not processed by the APT), `get()` returns `null` and the runtime falls back to standard Java reflection.

### Reflection fallback

The reflection fallback is slower but always correct. It is active when:

- The `jreactive-apt` processor was not run (e.g., in tests that do not invoke the full build).
- A component was added after the last compilation.
- The accessor class failed to load.

## Registering a custom accessor

You can manually register an accessor for testing or for components that are not processed by the APT:

```java
AccessorRegistry.register(MyComponent.class, new ComponentAccessor<MyComponent>() {

    @Override
    public void write(MyComponent c, String property, Object value) {
        if ("count".equals(property)) c.count = (Integer) value;
    }

    @Override
    public Object read(MyComponent c, String property) {
        if ("count".equals(property)) return c.count;
        return null;
    }

    @Override
    public Object call(MyComponent c, String method, Object... args) {
        if ("increment".equals(method)) { c.increment(); return null; }
        return null;
    }
});
```
