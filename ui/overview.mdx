The `jreactive-ui` library ships with nine pre-built components that integrate directly into any JReactive application. Each component is a standard `HtmlComponent` subclass with `@Bind` fields you set from Java — no JavaScript configuration needed.

## Maven dependency

```xml pom.xml
<dependency>
    <groupId>com.ciro</groupId>
    <artifactId>jreactive-ui</artifactId>
    <version>0.0.1-SNAPSHOT</version>
</dependency>
```

<Note>
  All JReactive artifacts are distributed via [JitPack](https://jitpack.io). Make sure the JitPack repository is declared in your `pom.xml` before adding this dependency. See the [Quickstart](/quickstart) for full repository configuration.
</Note>

## How component binding works

All `jreactive-ui` components use `@Bind` fields. When you place a component tag in your template, you pass values using standard attribute syntax or the reactive `:field="expr"` binding syntax:

```html template
<!-- Static string -->
<JButton label="Save" type="primary" onClick="save()" />

<!-- Reactive binding: evaluates the expression against server state -->
<JButton :label="buttonLabel" :disabled="isSaving" onClick="save()" />
```

On the Java side you set defaults or update them from `@Call` methods:

```java
public class MyPage extends AppPage {

    @State public boolean isSaving = false;
    @Bind public String buttonLabel = "Save";

    @Call
    public void save() {
        this.isSaving = true;
        // ... perform work ...
        this.isSaving = false;
    }
}
```

## Package

All components are in the `com.ciro.jreactive` package. Import them with:

```java
import com.ciro.jreactive.JButton;
import com.ciro.jreactive.JInput;
import com.ciro.jreactive.JSelect;
// etc.
```
